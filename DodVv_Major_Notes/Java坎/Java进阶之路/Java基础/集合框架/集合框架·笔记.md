## 一、集合体系总览

```
Collection（单列集合）
  ├── List（有序、可重复、有索引）
  │     ├── ArrayList    底层：数组    查询快、增删慢
  │     ├── LinkedList   底层：双向链表 增删快、查询慢
  │     └── Vector       线程安全（已过时）
  │
  └── Set（无序、不可重复、无索引）
        ├── HashSet       底层：哈希表
        ├── LinkedHashSet 底层：哈希表+链表（保持插入顺序）
        └── TreeSet       底层：红黑树（自动排序）

Map（双列集合：键值对）
  ├── HashMap        键无序，底层：数组+链表+红黑树
  ├── LinkedHashMap  键有序（插入顺序）
  └── TreeMap        键可排序（红黑树）
```

## 二、List 集合

### ArrayList（重点）

**特点：** 底层是 `Object[]` 数组，查询快（O(1)），增删慢（需要移动元素）。

**扩容机制：**
```java
// 无参构造：初始容量为0，第一次添加时扩容到10
// 每次扩容：newCapacity = oldCapacity + (oldCapacity >> 1)（1.5倍）
// 最大容量：Integer.MAX_VALUE - 8
```

**常用方法：**
```java
ArrayList<String> list = new ArrayList<>();
list.add("Java");           // 尾部添加
list.add(0, "Hello");       // 指定位置插入
list.get(0);                // 获取
list.set(0, "World");       // 修改
list.remove(0);             // 按索引删除
list.remove("Java");        // 按对象删除
list.size();                // 大小
list.contains("Java");      // 是否包含
```

### ArrayList vs LinkedList

| 对比 | ArrayList | LinkedList |
|:----|:----------|:-----------|
| 底层 | 数组 | 双向链表 |
| 随机访问（get） | O(1) **快** | O(n) 慢 |
| 头部插入/删除 | O(n) 慢 | O(1) **快** |
| 尾部插入/删除 | O(1) 快（扩容时O(n)） | O(1) 快 |
| 内存占用 | 少 | 多（需要存储前后指针） |

## 三、Set 集合

| 实现类 | 特点 | 底层 |
|:------|:-----|:-----|
| `HashSet` | 无序，最快 | HashMap（键即是元素） |
| `LinkedHashSet` | **有序**（插入顺序） | LinkedHashMap |
| `TreeSet` | **可排序**（默认升序） | TreeMap（红黑树） |

**HashSet 去重原理：**
1. 先调用 `hashCode()` 计算哈希值
2. 如果哈希值相同，再调用 `equals()` 比较
3. 只有两个都相同才视为重复元素

> **⚠️ 存储自定义对象到 HashSet 时，必须重写 `hashCode()` 和 `equals()` 才能正确去重！**

## 四、Map 集合（重点：HashMap）

### HashMap 的核心原理

**底层结构：** 数组 + 链表 + 红黑树

```
JDK7：数组 + 链表（头插法）
JDK8+：数组 + 链表 + 红黑树（尾插法）
```

**核心参数：**
| 参数 | 说明 | 默认值 |
|:----|:-----|:------:|
| `capacity` | 数组容量（2的幂） | 16 |
| `loadFactor` | 负载因子 | 0.75 |
| `threshold` | 阈值（capacity × loadFactor） | 12 |
| `TREEIFY_THRESHOLD` | 链表转红黑树阈值 | 8 |
| `UNTREEIFY_THRESHOLD` | 红黑树转链表阈值 | 6 |

**put 流程：**
1. 计算 key 的 hashCode，经过扰动函数后得到桶索引
2. 如果该位置为空，直接插入
3. 如果不为空，遍历链表/红黑树
4. 如果找到相同 key，覆盖旧值
5. 如果没找到，尾插法插入新节点
6. 如果链表长度 ≥ 8，转成红黑树
7. 如果 size > threshold，扩容为原来的2倍

**常用方法：**
```java
Map<String, Integer> map = new HashMap<>();
map.put("key", 1);           // 添加/覆盖
map.get("key");              // 获取（不存在返回null）
map.getOrDefault("key", 0);  // 获取，不存在返回默认值
map.remove("key");           // 删除
map.containsKey("key");      // 是否包含键
map.containsValue(1);        // 是否包含值
map.keySet();                // 获取所有键
map.values();                // 获取所有值
map.entrySet();              // 获取所有键值对
```

**三种遍历方式：**
```java
// 1. 键找值
for (String key : map.keySet()) {
    System.out.println(key + "=" + map.get(key));
}

// 2. 键值对
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + "=" + entry.getValue());
}

// 3. Lambda
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

## 五、Collections 工具类

```java
List<Integer> list = new ArrayList<>();
Collections.addAll(list, 3, 1, 2);   // 批量添加
Collections.sort(list);               // 升序排序
Collections.shuffle(list);            // 打乱顺序
Collections.reverse(list);            // 反转
Collections.max(list);                // 最大值
Collections.min(list);                // 最小值
Collections.binarySearch(list, 2);    // 二分查找（先排序）
```
