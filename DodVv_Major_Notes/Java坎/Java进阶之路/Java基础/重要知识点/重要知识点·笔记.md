## 一、泛型（Generic）

### 1. 为什么需要泛型？

在没有泛型之前，集合可以存储任意类型对象，取出时需要强制转换，容易产生 `ClassCastException`。

```java
// 无泛型时代码（不安全）
List list = new ArrayList();
list.add("Hello");
list.add(123);
String s = (String) list.get(1);  // 运行时异常！

// 有泛型（安全）
List<String> list = new ArrayList<>();
list.add("Hello");
// list.add(123);  // 编译报错
String s = list.get(0);  // 不需要强转
```

### 2. 泛型类

```java
public class Box<T> {
    private T item;
    
    public void set(T item) { this.item = item; }
    public T get() { return item; }
}

// 使用
Box<String> box = new Box<>();
box.set("二哥的Java进阶之路");
String content = box.get();
```

### 3. 泛型方法

```java
public <T> T getValue(T value) {
    return value;
}
```

### 4. 泛型通配符

| 通配符 | 含义 | 示例 |
|:-------|:-----|:-----|
| `?` | 任意类型 | `List<?>` |
| `? extends T` | T 或 T 的子类（上界） | `List<? extends Number>` |
| `? super T` | T 或 T 的父类（下界） | `List<? super Integer>` |

**PECS 原则（重点）：**
- `? extends T`：适合**读取**（Producer Extends）
- `? super T`：适合**写入**（Consumer Super）

### 5. 类型擦除

泛型信息只在**编译阶段**有效，运行时会进行类型擦除：

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
System.out.println(list1.getClass() == list2.getClass());  // true
// 运行时都是 ArrayList.class，泛型信息丢失
```

## 二、注解（Annotation）

### 1. 元注解

| 注解 | 作用 | 取值 |
|:----|:-----|:-----|
| `@Target` | 指定注解的使用位置 | METHOD、FIELD、TYPE、PARAMETER... |
| `@Retention` | 指定注解的保留策略 | SOURCE、CLASS、RUNTIME |
| `@Documented` | 是否包含在 Javadoc 中 | — |
| `@Inherited` | 是否被继承 | — |

### 2. 自定义注解

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyTest {
    String value() default "";
    int timeout() default 0;
}
```

### 3. 常见内置注解

| 注解 | 用途 |
|:----|:-----|
| `@Override` | 检查方法是否重写父类方法 |
| `@Deprecated` | 标记已过时 |
| `@SuppressWarnings` | 抑制编译器警告 |
| `@FunctionalInterface` | 标记函数式接口 |

## 三、反射（Reflection）

### 1. 获取 Class 对象的三种方式

```java
// 方式一：Class.forName()
Class<?> clazz1 = Class.forName("java.lang.String");

// 方式二：类名.class
Class<String> clazz2 = String.class;

// 方式三：对象.getClass()
String s = "hello";
Class<? extends String> clazz3 = s.getClass();
```

### 2. 反射的核心 API

```java
Class<?> clazz = User.class;

// 获取构造器
Constructor<?> constructor = clazz.getConstructor(String.class, int.class);
Object obj = constructor.newInstance("二哥", 18);

// 获取方法
Method method = clazz.getMethod("getName");
String name = (String) method.invoke(obj);

// 获取字段
Field field = clazz.getDeclaredField("name");
field.setAccessible(true);  // 绕过 private 限制
field.set(obj, "沉默王二");

// 获取注解
Annotation[] annotations = clazz.getAnnotations();
```

### 3. 反射的应用场景

- Spring 的 IoC 容器（通过反射创建 Bean）
- MyBatis 的 Mapper 映射
- 各种框架的依赖注入
- 动态代理（JDK 动态代理基于反射）

## 四、枚举（Enum）

```java
public enum Color {
    RED("红色"), GREEN("绿色"), BLUE("蓝色");
    
    private String name;
    
    Color(String name) {
        this.name = name;
    }
    
    public String getName() { return name; }
}

// 使用
Color c = Color.RED;
System.out.println(c.getName());  // "红色"
System.out.println(c.ordinal());  // 0（枚举常量的序号）
```
