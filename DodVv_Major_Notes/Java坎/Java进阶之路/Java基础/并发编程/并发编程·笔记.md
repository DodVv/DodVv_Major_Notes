# Java并发编程

> 来源：[并发编程学习路线](https://javabetter.cn/xuexiluxian/java/thread.html) | [并发编程](https://javabetter.cn/thread/wangzhe-thread.html)

## 一、线程基础

### 创建线程的三种方式

```java
// 1. 继承 Thread
class MyThread extends Thread {
    @Override
    public void run() { System.out.println("线程1"); }
}
new MyThread().start();

// 2. 实现 Runnable（推荐）
new Thread(() -> System.out.println("线程2")).start();

// 3. 实现 Callable（有返回值）
FutureTask<String> task = new FutureTask<>(() -> "线程3");
new Thread(task).start();
String result = task.get();  // 获取返回值
```

### 线程生命周期

```
新建（NEW）→ 就绪（RUNNABLE）→ 运行 → 死亡（TERMINATED）
                ↕            ↕
            等待（WAITING）  阻塞（BLOCKED）
```

| 状态 | 含义 |
|:----|:-----|
| NEW | 创建了 Thread 对象，还没调用 start() |
| RUNNABLE | 调用了 start()，正在运行或等待 CPU 调度 |
| BLOCKED | 等待获取锁（synchronized 没抢到） |
| WAITING | 调用了 wait()、join()、park() |
| TIMED_WAITING | 调用了 sleep(time)、wait(time) |
| TERMINATED | run() 执行完毕 |

## 二、synchronized 关键字

### 使用方式

```java
// 1. 修饰实例方法（锁的是当前对象）
public synchronized void increment() { count++; }

// 2. 修饰静态方法（锁的是类对象）
public static synchronized void decrement() { count--; }

// 3. 修饰代码块（更灵活）
public void add() {
    synchronized (this) { count++; }        // 锁对象
    synchronized (MyClass.class) { ... }    // 锁类
    synchronized (lockObj) { ... }          // 锁任意对象
}
```

### synchronized 底层原理

- **JDK6 之前**：重量级锁（依赖操作系统 Mutex Lock）
- **JDK6 之后**：锁升级（无锁 → 偏向锁 → 轻量级锁 → 重量级锁）

> **锁升级（优化）：** 先偏向锁（一个线程访问）→ 轻量级锁（自旋）→ 重量级锁（阻塞）

## 三、volatile 关键字

**volatile 的两大作用：**
1. **保证可见性**：一个线程修改变量，其他线程立即看到
2. **禁止指令重排序**：防止 JVM 优化时重排指令

```java
private volatile boolean flag = false;
// 线程1：flag = true;  // 立即对其他线程可见
// 线程2：if (flag) { ... }  // 立即看到线程1的修改
```

> **volatile 不保证原子性！** 比如 `count++` 这种复合操作，还需要加锁。

## 四、JUC 并发工具

### Lock 接口

```java
Lock lock = new ReentrantLock();
lock.lock();
try {
    // 临界区代码
} finally {
    lock.unlock();  // 必须在 finally 中释放
}
```

### 线程池（ExecutorService）

**创建线程池的三种方式：**

```java
// 1. 固定大小线程池（常用）
ExecutorService pool = Executors.newFixedThreadPool(5);

// 2. 缓存线程池
ExecutorService pool = Executors.newCachedThreadPool();

// 3. 单线程池
ExecutorService pool = Executors.newSingleThreadExecutor();

// 4. 定时任务线程池
ScheduledExecutorService pool = Executors.newScheduledThreadPool(3);
```

**提交任务：**
```java
pool.execute(() -> System.out.println("无返回值"));     // Runnable
Future<String> future = pool.submit(() -> "有返回值");   // Callable
```

**推荐用 ThreadPoolExecutor 手动创建（更可控）：**
```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    2,              // corePoolSize（核心线程数）
    5,              // maximumPoolSize（最大线程数）
    60,             // keepAliveTime
    TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(100),  // 任务队列
    new ThreadPoolExecutor.AbortPolicy()  // 拒绝策略
);
```

**线程池的 4 种拒绝策略：**
| 策略 | 说明 |
|:----|:-----|
| AbortPolicy（默认） | 直接抛异常 |
| CallerRunsPolicy | 调用者线程自己执行 |
| DiscardPolicy | 直接丢弃任务 |
| DiscardOldestPolicy | 丢弃队列中最旧的任务 |

## 五、常见并发容器

| 容器 | 对应的非并发版本 | 特点 |
|:----|:----------------|:-----|
| `ConcurrentHashMap` | HashMap | 分段锁/CAS，高并发安全 |
| `CopyOnWriteArrayList` | ArrayList | 读不加锁，写时复制 |
| `CopyOnWriteArraySet` | HashSet | 基于CopyOnWriteArrayList |
| `BlockingQueue` | Queue | 阻塞队列（生产者-消费者模式） |
| `ConcurrentLinkedQueue` | LinkedList | 无锁、CAS实现 |

## 六、AQS（AbstractQueuedSynchronizer）

**AQS 是 JUC 的基石**，ReentrantLock、Semaphore、CountDownLatch 等底层都基于 AQS。

**AQS 核心思想：**
- 一个 `volatile int state` 表示同步状态
- 一个 CLH 双向队列存储等待的线程
- 定义了独占模式和共享模式
