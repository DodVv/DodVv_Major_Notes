## MySQL

### 索引
| 索引类型 | 特点 |
|:---------|:-----|
| B+Tree 索引（InnoDB默认） | 多路平衡树，叶子节点存数据 |
| Hash 索引 | 等值查询快，不支持范围查询 |
| 聚簇索引 | 数据和索引在一起（InnoDB主键） |
| 辅助索引 | 存储主键值，回表查询 |
| 联合索引 | 最左前缀原则 |

### 事务
- **ACID**：原子性、一致性、隔离性、持久性
- **事务隔离级别**：读未提交 → 读已提交 → 可重复读（MySQL默认）→ 串行化
- **MVCC**：多版本并发控制，实现"读已提交"和"可重复读"

## Redis

### 五大数据结构
| 类型 | 说明 | 常用命令 |
|:----|:-----|:---------|
| String | 字符串 | GET/SET/INCR |
| List | 列表（链表） | LPUSH/RPOP/LRANGE |
| Set | 无序集合 | SADD/SMEMBERS/SINTER |
| ZSet | 有序集合 | ZADD/ZRANGE/ZSCORE |
| Hash | 键值对 | HSET/HGET/HGETALL |

### 缓存三大问题
| 问题 | 现象 | 解决 |
|:----|:-----|:-----|
| **缓存穿透** | 查不存在的数据，穿透到 DB | 布隆过滤器 / 缓存空值 |
| **缓存击穿** | 热点 key 过期，大并发打到 DB | 互斥锁 / 逻辑过期 |
| **缓存雪崩** | 大量 key 同时过期 | 过期时间随机化 / 多级缓存 |
