## 一、JVM 内存区域（运行时数据区）

```
┌──────────────────────────────────────┐
│              线程共享                  │
│  ┌────────────────────────────────┐  │
│  │         方法区（Metaspace）      │  │
│  │  类信息、常量、静态变量、JIT代码  │  │
│  └────────────────────────────────┘  │
│  ┌────────────────────────────────┐  │
│  │           堆（Heap）            │  │
│  │    几乎所有对象实例都在此分配     │  │
│  └────────────────────────────────┘  │
├──────────────────────────────────────┤
│              线程私有                 │
│  ┌─────────────┐  ┌───────────────┐  │
│  │  虚拟机栈    │  │  本地方法栈    │  │
│  │ (Java方法)   │  │ (native方法)  │  │
│  └─────────────┘  └───────────────┘  │
│  ┌────────────────────────────────┐  │
│  │        程序计数器（PC）         │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 各区域详解

| 区域 | 共享？ | 作用 | 异常 |
|:----|:-----:|:-----|:-----|
| **堆（Heap）** | ✅ 共享 | 存储所有对象实例和数组 | OutOfMemoryError |
| **方法区（Metaspace）** | ✅ 共享 | JDK8后元空间取代永久代，存储类信息、常量、静态变量 | OutOfMemoryError |
| **虚拟机栈** | ❌ 私有 | 每个方法调用对应一个栈帧，存储局部变量表、操作数栈等 | StackOverflowError |
| **本地方法栈** | ❌ 私有 | 为 native 方法服务 | StackOverflowError |
| **程序计数器** | ❌ 私有 | 记录当前线程执行的字节码行号 | 无 |

## 二、堆内存分代

```
┌────────────────────────────────────────────────┐
│                   堆（Heap）                     │
│  ┌──────────────┬────────┬──────────────────┐   │
│  │    Eden      │  S0    │  S1              │   │
│  │   (80%)      │  (10%) │   (10%)          │   │
│  ├────── 新生代 ──────┘     └────────────────┤   │
│  │                                            │   │
│  │             老年代（Old Gen）               │   │
│  │        存储长期存活的对象                    │   │
│  └────────────────────────────────────────────┘  │
│             元空间（Metaspace）                   │
│          （JDK8+，本地内存）                      │
└──────────────────────────────────────────────────┘
```

### 对象分配与晋升

1. 新对象优先分配在 **Eden 区**
2. 第一次 Minor GC 后存活的对象进入 **S0（Survivor）**
3. 再次 Minor GC，S0 存活对象进入 **S1**（交换）
4. 年龄达到阈值（默认15）或放不下时，进入**老年代**

## 三、垃圾回收（GC）

### 如何判断对象可回收？

| 算法 | 原理 | 说明 |
|:----|:-----|:-----|
| **引用计数法** | 每个对象有引用计数器，为0可回收 | Python 用，Java 不用（有循环引用问题） |
| **可达性分析** | 从 GC Roots 出发，不可达的对象可回收 | **Java 采用的方案** |

**GC Roots 包括：**
- 虚拟机栈中引用的对象
- 方法区中静态属性引用的对象
- 方法区中常量引用的对象
- 本地方法栈 JNI 引用的对象

### 垃圾收集算法

| 算法 | 原理 | 适用 |
|:----|:-----|:-----|
| **标记-清除** | 标记→清除，有碎片 | 老年代 |
| **标记-复制** | 分成两块，存活复制到另一块，整块清除 | **新生代** |
| **标记-整理** | 标记→存活对象往一端移动 | **老年代** |

### 常用垃圾收集器

| 收集器 | 作用区域 | 算法 | 特点 |
|:-------|:--------:|:-----|:-----|
| **G1**（JDK9+默认） | 新生代+老年代 | 分区+复制 | 低延迟、可预测停顿 |
| **CMS** | 老年代 | 标记-清除 | 并发收集、低停顿、有碎片 |
| **Parallel** | 新生代 | 复制 | 高吞吐量、适合后台计算 |
| **Serial** | 新生代 | 复制 | 单线程、Client默认 |
| **ZGC**（JDK11+） | 全部 | 并发标记-整理 | 超低延迟（<10ms） |

## 四、类加载机制

### 类加载过程

```
加载（Loading）→ 验证（Verification）→ 准备（Preparation）→ 解析（Resolution）
→ 初始化（Initialization）
```

### 类加载器

```
Bootstrap ClassLoader（启动类加载器）
  ↑ 委托 ↓
Extension ClassLoader（扩展类加载器）
  ↑ 委托 ↓
Application ClassLoader（应用类加载器）
  ↑ 委托 ↓
自定义类加载器
```

**双亲委派模型：** 类加载器收到类加载请求时，先委派给父类加载器加载，只有当父类加载器无法完成时才自己加载。

**双亲委派的好处：**
- 避免核心 API 被篡改（如自己写个 `java.lang.String` 不会被加载）
- 保证 Java 核心库的安全性

## 五、JVM 调优常见参数

```bash
# 堆内存
-Xms512m          # 初始堆大小
-Xmx1024m         # 最大堆大小
-Xmn256m          # 新生代大小
-XX:MetaspaceSize=128m  # 元空间大小

# GC 选择
-XX:+UseG1GC           # 使用 G1 收集器
-XX:+UseParallelGC     # 使用 Parallel 收集器

# 调试信息
-XX:+PrintGCDetails    # 打印 GC 日志
-XX:+HeapDumpOnOutOfMemoryError  # OOM 时生成堆转储
```
