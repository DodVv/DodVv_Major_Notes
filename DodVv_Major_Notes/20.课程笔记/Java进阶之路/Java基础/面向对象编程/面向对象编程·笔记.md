# Java面向对象编程

## 一、面向对象三大特性

### 1. 封装（Encapsulation）

**核心思想**：隐藏内部实现细节，对外提供公共访问接口。

```java
public class Person {
    // 私有成员变量
    private String name;
    private int age;

    // 公共 getter/setter
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    
    public int getAge() { return age; }
    public void setAge(int age) {
        // 可以在 setter 中加入校验逻辑
        if (age > 0 && age < 150) {
            this.age = age;
        }
    }
}
```

**封装的好处：**
- 保护数据安全，防止外部随意修改
- 可在 setter 中加入校验逻辑
- 便于后期维护和修改内部实现

### 2. 继承（Inheritance）

**使用 `extends` 关键字**，子类继承父类的非私有成员。

```java
// 父类
public class Animal {
    protected String name;
    public void eat() { System.out.println("吃东西"); }
}

// 子类
public class Dog extends Animal {
    public void bark() { System.out.println("汪汪汪"); }
}

// 使用
Dog dog = new Dog();
dog.name = "旺财";
dog.eat();   // 继承自父类
dog.bark();  // 子类自己的方法
```

**继承的特点：**
- Java 是**单继承**（一个子类只能有一个直接父类）
- 子类可以继承父类的 `public` 和 `protected` 成员
- 子类无法继承父类的 `private` 成员
- 构造方法不能被继承
- 可以使用 `super` 关键字访问父类成员

**方法重写（Override）：**
```java
@Override  // 注解，检查是否正确重写
public void eat() {
    super.eat();  // 调用父类方法
    System.out.println("狗啃骨头");
}
```

### 3. 多态（Polymorphism）

**多态的三种形式：**

| 形式 | 示例 |
|:----|:-----|
| 对象多态 | `Animal a = new Dog();` |
| 行为多态 | `a.eat();` → 执行的是子类的 eat() |

**多态的访问规则：**
```java
Animal a = new Dog();
a.eat();      // ✅ 方法：编译看左边，运行看右边（调用子类重写的方法）
// a.bark();  // ❌ 编译报错：父类没有 bark() 方法
```

**类型转换：**
```java
// 向下转型（找回子类特有方法）
if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();
}
```

> **instanceof 的作用**：判断一个对象是否是某个类的实例，避免强制类型转换时出现 `ClassCastException`。

## 二、抽象类

```java
// 抽象类：使用 abstract 关键字
public abstract class Shape {
    protected String color;
    
    // 抽象方法：没有方法体，子类必须重写
    public abstract double getArea();
    
    // 普通方法：可以有实现
    public void setColor(String color) {
        this.color = color;
    }
}

public class Circle extends Shape {
    private double radius;
    
    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }
}
```

**抽象类的特点：**
- 抽象类不能直接 `new`
- 可以有构造方法、成员变量、普通方法
- 子类必须重写所有抽象方法（除非子类也是抽象类）

## 三、接口（Interface）

```java
// 定义接口
public interface Flyable {
    // 抽象方法（默认 public abstract）
    void fly();
    
    // JDK8：默认方法（可以有方法体）
    default void glide() {
        System.out.println("滑翔中...");
    }
    
    // JDK8：静态方法
    static void info() {
        System.out.println("飞行接口");
    }
}

// 实现接口（implements）
public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
}
```

**接口 vs 抽象类：**

| 对比项 | 抽象类 | 接口 |
|:------|:-------|:-----|
| 关键字 | `abstract class` | `interface` |
| 继承方式 | `extends`（单继承） | `implements`（多实现） |
| 成员变量 | 任意类型 | `public static final` 常量 |
| 构造方法 | ✅ 可以有 | ❌ 不能有 |
| 方法 | 抽象 + 普通方法 | 抽象 + default + static 方法 |
| 设计思想 | "是什么"（is-a） | "能做什么"（like-a） |

## 四、内部类

| 类型 | 定义位置 | 特点 |
|:----|:---------|:-----|
| 成员内部类 | 类中方法外 | 可以访问外部类的私有成员 |
| 静态内部类 | 用 `static` 修饰 | 不依赖外部类实例 |
| 局部内部类 | 方法中 | 作用域仅限于该方法 |
| 匿名内部类 | 在创建对象时定义 | 适用于一次性使用的实现 |

**匿名内部类（常用）：**
```java
// 常见用法：创建线程
Thread t = new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("匿名内部类");
    }
});

// Lambda 简写（JDK8+）
Thread t = new Thread(() -> System.out.println("Lambda"));
```
