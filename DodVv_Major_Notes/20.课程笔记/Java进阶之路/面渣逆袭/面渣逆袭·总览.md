## Java基础篇

### 1. 面向对象三大特性
**封装**：隐藏内部细节，对外提供公共接口。通过 `private` + getter/setter 实现。

**继承**：子类继承父类，使用 `extends`。Java 单继承。子类可重写父类方法。

**多态**：父类引用指向子类对象。方法调用看实际类型（运行期），变量访问看声明类型（编译期）。

### 2. == 和 equals() 的区别
- `==`：基本类型比较值，引用类型比较**地址**
- `equals()`：默认比较地址，但 String、Integer 等类已重写为比较**内容**

```java
String s1 = new String("abc");
String s2 = new String("abc");
s1 == s2;          // false
s1.equals(s2);     // true
```

### 3. final 关键字
| 修饰 | 效果 |
|:-----|:-----|
| `final class` | 不能被继承（如 String） |
| `final method` | 不能被重写 |
| `final variable` | 引用不能变（但对象内容可变） |

### 4. String、StringBuilder、StringBuffer 的区别
| 对比 | String | StringBuilder | StringBuffer |
|:----|:-------|:--------------|:-------------|
| 可变性 | 不可变 | 可变 | 可变 |
| 线程安全 | 安全（不可变） | 不安全 | 安全（synchronized） |
| 性能 | 慢（拼接创建新对象） | 快 | 较慢 |
| 使用场景 | 少量字符串 | 字符串大量拼接（单线程） | 字符串大量拼接（多线程） |

### 5. 重载（Overload）和重写（Override）的区别
| 对比 | 重载 | 重写 |
|:----|:-----|:-----|
| 发生位置 | 同一个类中 | 父子类中 |
| 方法名 | 相同 | 相同 |
| 参数列表 | **必须不同** | 必须相同 |
| 返回值 | 无关 | 相同或子类 |
| 关键字 | 无 | `@Override` |

## 集合框架篇

### 1. ArrayList 和 LinkedList 的区别
**ArrayList**：基于数组，查询快 O(1)，增删慢 O(n)（需移动元素）
**LinkedList**：基于双向链表，查询慢 O(n)，头尾增删快 O(1)

### 2. HashMap 的底层原理
**JDK7**：数组 + 链表（头插法）
**JDK8+**：数组 + 链表 + 红黑树（尾插法）

**put 流程：** 计算 hashCode → 求桶索引 → 若为空直接插入 → 若不为空遍历链表/红黑树 → 找到相同 key 覆盖 → 否则插入 → 链表≥8 转红黑树 → size>阈值 扩容2倍

### 3. ConcurrentHashMap 如何保证线程安全？
**JDK7**：分段锁（Segment 数组，继承 ReentrantLock）
**JDK8+**：`synchronized` + CAS（锁链表头节点，粒度更细）

### 4. HashSet 如何保证元素不重复？
底层是 HashMap（键即元素，值为常量 `PRESENT`），通过 `hashCode()` + `equals()` 判断重复：

1. 先计算元素的 `hashCode()`
2. 找到对应桶位置
3. 如果桶内已有元素，用 `equals()` 比较
4. 如果 `equals()` 返回 true，视为重复，不添加

## 并发编程篇

### 1. synchronized 和 ReentrantLock 的区别
| 对比 | synchronized | ReentrantLock |
|:----|:------------|:--------------|
| 关键字/类 | JVM 关键字 | JDK 提供的类 |
| 锁释放 | 自动释放 | 必须手动 unlock() |
| 功能 | 少 | 多（可中断、可定时、公平锁） |
| 底层 | 锁升级（偏向→轻量→重量） | AQS + CAS |
| 性能 | JDK6 后与 Lock 相近 | 优秀 |

### 2. volatile 的作用
1. **保证可见性**：一个线程修改，其他线程立即可见
2. **禁止指令重排序**：防止 JVM 优化时重排指令
3. **不保证原子性**：`count++` 仍需加锁

### 3. 线程池的 7 个核心参数
```java
ThreadPoolExecutor(int corePoolSize,      // 核心线程数
                   int maximumPoolSize,   // 最大线程数
                   long keepAliveTime,    // 空闲线程存活时间
                   TimeUnit unit,         // 时间单位
                   BlockingQueue<Runnable> workQueue,  // 任务队列
                   ThreadFactory threadFactory,        // 线程工厂
                   RejectedExecutionHandler handler)   // 拒绝策略
```

## JVM 篇

### 1. JVM 内存区域
- **堆**：对象实例，GC 主要区域
- **方法区（元空间）**：类信息、常量、静态变量
- **虚拟机栈**：栈帧（局部变量表、操作数栈）
- **本地方法栈**：native 方法
- **程序计数器**：字节码行号

### 2. GC 判断对象可回收
- **可达性分析**：从 GC Roots 出发，不可达的对象可回收
- GC Roots：栈引用、静态引用、JNI 引用

### 3. 常见的垃圾收集器
- **G1（JDK9默认）**：低延迟，可预测停顿
- **CMS**：并发收集，低停顿
- **Parallel**：高吞吐，适合后台计算
- **ZGC（JDK11）**：超低延迟 <10ms

### 4. 类加载的双亲委派机制
类加载器先委派给父加载器加载，父无法加载时才自己加载。**保证核心 API 不被篡改。**
