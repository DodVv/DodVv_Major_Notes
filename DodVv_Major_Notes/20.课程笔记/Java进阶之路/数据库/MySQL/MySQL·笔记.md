## 二、索引

### B+Tree 索引结构
- 非叶子节点只存索引键（不存数据），可以存放更多索引
- 叶子节点存放完整数据，且用链表连接（适合范围查询）
- 高度通常为 3-4 层，千万级数据查询只要 3-4 次 IO

### 聚簇索引 vs 辅助索引
| 索引类型 | 说明 | 叶子节点 |
|:---------|:-----|:---------|
| 聚簇索引 | 主键索引 | 存储完整行数据 |
| 辅助索引 | 非主键索引 | 存储主键值 |

**回表查询**：通过辅助索引找到主键，再通过聚簇索引找到完整数据

### 最左前缀原则
联合索引 `(a, b, c)` 能用到索引的情况：
- `where a = 1` ✅
- `where a = 1 and b = 2` ✅
- `where a = 1 and b = 2 and c = 3` ✅
- `where b = 2` ❌（没用到 a）

## 三、事务与 MVCC

### MVCC 核心组件
| 组件 | 说明 |
|:----|:-----|
| 隐藏字段 | DB_TRX_ID（事务ID）、DB_ROLL_PTR（回滚指针） |
| undo log | 记录数据修改前的版本，用于回滚和 MVCC |
| ReadView | 事务执行时看到的快照 |

**MVCC 解决不可重复读**（RR级别）：同一事务中，每次 read 使用的 ReadView 相同，看到的数据版本不变。

## 四、SQL 优化

1. **查询 SQL 尽量走索引**：`explain` 查看执行计划
2. **避免 SELECT ***：只查询需要的字段
3. **避免索引失效**：不在索引列上使用函数、类型转换、LIKE '%xx'
4. **分页优化**：大偏移量时用子查询优化
5. **合理设计字段类型**：能用 `int` 不用 `varchar`
