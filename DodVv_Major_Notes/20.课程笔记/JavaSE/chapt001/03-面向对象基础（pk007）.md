## 💻 核心代码示例

### 1. 类与对象快速入门

```java
// ===== Student.java（类：对象的模板）=====
public class Student {
    // 成员变量（对象的属性）
    String name;
    double chinese;
    double math;

    // 成员方法（对象的行为）
    public void printTotalScore() {
        // 在方法内可以直接访问成员变量
        System.out.println(name + "的总成绩是：" + (chinese + math));
    }

    public void printAverageScore() {
        System.out.println(name + "的平均成绩是：" + (chinese + math) / 2.0);
    }
}
```

```java
// ===== Test.java（测试类）=====
public class Test {
    public static void main(String[] args) {
        // 1、创建对象：类名 对象名 = new 类名();
        Student s1 = new Student();
        
        // 2、给对象赋值：对象名.成员变量 = 值;
        s1.name = "播妞";
        s1.chinese = 100;
        s1.math = 100;
        
        // 3、调用对象的方法：对象名.成员方法();
        s1.printTotalScore();   // 输出：播妞的总成绩是：200.0
        s1.printAverageScore(); // 输出：播妞的平均成绩是：100.0

        // 4、可以创建多个对象，数据互不影响
        Student s2 = new Student();
        s2.name = "播仔";
        s2.chinese = 59;
        s2.math = 100;
        
        System.out.println(s1);  // 输出地址值，如：cn.itcast.Student@4eec7777
        System.out.println(s2);  // 不同对象，地址不同
    }
}
```

### 2. 类与对象注意事项

```java
public class Test2 {
    public static void main(String[] args) {
        // ⭐ 1、成员变量有默认值，不需要手动赋初始值
        Student s = new Student();
        System.out.println(s.name);    // null
        System.out.println(s.chinese); // 0.0

        // ⭐ 2、一个代码文件中可以写多个class，但只能一个用public修饰，
        //      且public修饰的类名必须与文件名相同

        // ⭐ 3、对象与对象之间数据互不影响（除非指向同一个对象！）
        Student s1 = new Student();
        s1.name = "张三";
        
        Student s2 = s1;          // s2和s1指向同一个对象（地址赋值）
        s2.name = "李四";
        System.out.println(s1.name); // "李四"——被改了！
        
        // ⭐ 4、垃圾对象：没有变量引用的对象
        s1 = null;
        s2 = null;
        // System.out.println(s1.name); // NullPointerException！
    }
}
```

### 3. 构造器（Constructor）🔥

```java
// ===== Student.java =====
public class Student {
    String name;
    double score;

    // 【无参数构造器】
    public Student() {
        System.out.println("无参数构造器被触发了~");
    }

    // 【有参数构造器】（重载）
    public Student(String name, double score) {
        System.out.println("有参数构造器被触发了~~");
        this.name = name;    // this用于区分成员变量和参数
        this.score = score;
    }
}
```

```java
// ===== Test.java =====
public class Test {
    public static void main(String[] args) {
        // 调用有参构造器：创建对象的同时赋值
        Student s = new Student("播仔", 99);
        
        // 调用无参构造器
        Student s1 = new Student();
        s1.name = "播妞";   // 创建后再赋值
        s1.score = 100;
    }
}
```

> **⚠️ 老师强调**：如果你不写任何构造器，Java会默认送你一个无参构造器。但一旦你写了**有参构造器**，默认无参构造器就**没有了**！建议**两个都写上**。

### 4. 封装（Encapsulation）🔥🔥🔥

```java
// ===== Student.java =====
public class Student {
    // 核心原则：合理隐藏、合理暴露
    
    private double score;  // private：隐藏细节，只能在类内部访问

    // 对外提供公开的getter/setter方法
    public double getScore() {     // 读取数据
        return score;
    }

    public void setScore(double score) {  // 修改数据（可加入校验逻辑）
        if (score >= 0 && score <= 100) {
            this.score = score;
        } else {
            System.out.println("分数不合法！");
        }
    }

    private void printPass() {    // private方法：对外隐藏
        System.out.println(score >= 60 ? "及格" : "不及格");
    }
}
```

```java
// ===== Test.java =====
public class Test {
    public static void main(String[] args) {
        Student s1 = new Student();
        // s1.score = 99;      // ❌ 编译报错（private）
        s1.setScore(99);       // ✅ 通过公开方法访问
        System.out.println(s1.getScore());
    }
}
```

### 5. this 关键字 🔥

```java
public class Student {
    String name;
    double score;

    // this的本质：代表当前对象的地址（谁调用方法，this就指向谁）
    public void test(Student this) {  // this是JVM隐式传进来的
        System.out.println(this);     // 输出当前对象的地址
    }

    // ⭐ 核心用途：区分成员变量和局部变量（同名时）
    public void printPass(double score) {
        // score是参数（局部变量），this.score是成员变量
        if (this.score >= score) {
            System.out.println("考上了！");
        } else {
            System.out.println("没考过~~");
        }
    }
}
```

```java
// ===== Test.java =====
public class Test {
    public static void main(String[] args) {
        Student s1 = new Student();
        System.out.println(s1);         // 地址1
        s1.test();                      // 输出地址1（和上面相同）

        Student s2 = new Student();
        System.out.println(s2);         // 地址2
        s2.test();                      // 输出的this也是地址2

        // this就是：那个对象调用的方法，this就代表那个对象
    }
}
```

### 6. 成员变量 vs 局部变量 ⭐⭐⭐

```java
public class Student {
    // 【成员变量】（Field）—— 定义在类中、方法外
    String name;      // 有默认值：null
    double score;     // 有默认值：0.0

    public void aaa() {
        double score = 98;     // 【局部变量】—— 定义在方法中
                               // 没有默认值，必须初始化
        System.out.println(score);       // 98（就近原则：局部变量优先）
        System.out.println(this.score);  // 成员变量（用this区分）
    }
}
```

```java
// ===== 区别总结 =====
public class Test {
    public static void main(String[] args) {
        // 1、位置不同：成员变量(类中方法外) vs 局部变量(方法中/形参)
        
        // 2、默认值：成员变量有默认值 vs 局部变量无默认值
        int b = 20;  // 局部变量必须赋值
        
        // 3、内存位置：成员变量(堆) vs 局部变量(栈)
        
        // 4、作用域：成员变量(整个对象) vs 局部变量(所属的大括号内)
        
        // 5、生命周期：成员变量(与对象同生共死) vs 局部变量(方法调用而生、结束而亡)
    }
}
```

### 7. JavaBean 实体类规范 🔥🔥🔥

```java
// ===== Student.java（标准的JavaBean）=====
public class Student {
    // 规则1：成员变量使用 private 私有
    private String name;
    private double score;

    // 规则2：提供无参数构造器
    public Student() {
    }

    // 可提供有参数构造器（可选）
    public Student(String name, double score) {
        this.name = name;
        this.score = score;
    }

    // 规则3：为每个成员变量提供 getter/setter
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public double getScore() {
        return score;
    }

    public void setScore(double score) {
        this.score = score;
    }
}
```

```java
// ===== StudentOperator.java（操作类，对实体类进行业务处理）=====
public class StudentOperator {
    private Student student;  // 持有学生对象的引用

    // 构造器注入学生对象
    public StudentOperator(Student student) {
        this.student = student;

        // 通过getter获取数据，而不是直接访问
        if (student.getScore() >= 60) {
            System.out.println(student.getName() + "成绩及格");
        } else {
            System.out.println(student.getName() + "成绩不及格");
        }
    }
```

### 8. 电影管理系统（综合实战）🌟🌟🌟

**面向对象设计思路：** ① 实体类（只存数据）→ ② 业务操作类（处理数据）→ ③ 测试类（入口）

```java
// ===== Movie.java（实体类）=====
public class Movie {
    // private成员变量
    private int id;
    private String name;
    private double price;
    private double score;
    private String director;
    private String actor;
    private String info;

    // 无参构造器
    public Movie() {}
    
    // 有参构造器（快速创建对象并赋值）
    public Movie(int id, String name, double price, double score,
                  String director, String actor, String info) {
        this.id = id;
        this.name = name;
        this.price = price;
        this.score = score;
        this.director = director;
        this.actor = actor;
        this.info = info;
    }

    // 所有成员变量的getter/setter（alt+insert快捷键生成）
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    public String getName() { return name; }
    // ... 其他getter/setter略
}
```

```java
// ===== MovieOperator.java（业务操作类）=====
public class MovieOperator {
    private Movie[] movies;  // 持有电影数组

    // ⭐ 通过构造器把数据传递进来（面向对象：数据与操作分离）
    public MovieOperator(Movie[] movies) {
        this.movies = movies;
    }

    /** 1、展示全部电影信息 */
    public void printAllMovies() {
        for (int i = 0; i < movies.length; i++) {
            Movie m = movies[i];
            System.out.println("编号：" + m.getId());
            System.out.println("名称：" + m.getName());
            System.out.println("价格：" + m.getPrice());
            System.out.println("------------------------");
        }
    }

    /** 2、根据id查询电影详情 */
    public void searchMovieById(int id) {
        for (int i = 0; i < movies.length; i++) {
            Movie m = movies[i];
            if (m.getId() == id) {
                System.out.println("编号：" + m.getId());
                System.out.println("名称：" + m.getName());
                System.out.println("价格：" + m.getPrice());
                System.out.println("得分：" + m.getScore());
                System.out.println("导演：" + m.getDirector());
                System.out.println("主演：" + m.getActor());
                System.out.println("其他信息：" + m.getInfo());
                return;  // ⭐ 找到后立即结束方法
            }
        }
        System.out.println("没有该电影信息~");
    }
}
```

```java
// ===== Test.java（测试类：程序入口）=====
import java.util.Scanner;

public class Test {
    public static void main(String[] args) {
        // 1、准备电影数据（用数组存储多个Movie对象）
        Movie[] movies = new Movie[4];
        movies[0] = new Movie(1, "水门桥", 38.9, 9.8, "徐克", "吴京", "12万人想看");
        movies[1] = new Movie(2, "出拳吧", 39, 7.8, "唐晓白", "田雨", "3.5万人想看");
        movies[2] = new Movie(3, "月球陨落", 42, 7.9, "罗兰", "贝瑞", "17.9万人想看");
        movies[3] = new Movie(4, "一点就到家", 35, 8.7, "许宏宇", "刘昊然", "10.8万人想看");

        // 2、创建操作类对象，传入电影数据
        MovieOperator operator = new MovieOperator(movies);

        // 3、菜单系统：死循环 + switch
        Scanner sc = new Scanner(System.in);
        while (true) {
            System.out.println("==电影信息系统==");
            System.out.println("1、查询全部电影信息");
            System.out.println("2、根据id查询电影详情");
            System.out.print("请输入操作命令：");
            int command = sc.nextInt();
            switch (command) {
                case 1:
                    operator.printAllMovies();
                    break;
                case 2:
                    System.out.print("请输入电影id：");
                    int id = sc.nextInt();
                    operator.searchMovieById(id);
                    break;
                default:
                    System.out.println("命令有误！");
            }
        }
    }
}
```

## 🧩 知识点拆解

### 📌 面向对象核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| **类（Class）** | 对象的模板，定义属性+行为 | 图纸 |
| **对象（Object）** | 类的具体实例 | 根据图纸造出来的房子 |
| **封装** | 隐藏内部细节，对外提供公共接口 | 手机—按钮暴露、电路隐藏 |
| **构造器** | 创建对象时自动执行的特殊方法 | 造房子时打地基 |

### 📌 封装的设计规范

| 项目 | 内容 |
|------|------|
| **核心原则** | **合理隐藏、合理暴露** |
| **实现手段** | `private` 隐藏成员变量，`public getter/setter` 暴露访问通道 |
| **优点** | ①保护数据安全 ②可以在setter中加入校验逻辑 ③便于后期维护 |

> **老师强调**：封装是面向对象的**第一大特性**。写类的时候，上来就把成员变量全写 `private`，然后用 `alt+insert` 一键生成 getter/setter。

### 📌 构造器详解

| 类型 | 格式 | 特点 |
|------|------|------|
| **无参构造** | `public 类名() {}` | 不写时系统默认提供，写了有参构造后就没有了 |
| **有参构造** | `public 类名(参数) {}` | 创建对象的同时给成员变量赋值 |

> **💡 老师名言**："构造器就是用来创建对象的，不是用来写业务逻辑的。创建完对象就让它走人。"

### 📌 this 本质

| 本质 | 使用场景 |
|------|----------|
| **当前对象的地址** | ①区分同名成员变量和局部变量 ②构造器中给成员变量赋值 |

> **老师强调**：this是一个隐式参数，JVM在调用方法时自动传入。谁调用方法，this就指向谁。

### 📌 成员变量 vs 局部变量（完整对比）

| 对比项 | 成员变量 | 局部变量 |
|--------|----------|----------|
| **位置** | 类中方法外 | 方法内/形参/代码块 |
| **默认值** | ✅ 有（0/null/false） | ❌ 无，必须手动赋值 |
| **内存** | 堆（Heap） | 栈（Stack） |
| **作用域** | 整个对象 | 所属的大括号 |
| **生命周期** | 与对象同生共死 | 方法调用而生，结束而亡 |

### 📌 JavaBean 三大规范

| 规范 | 要求 | 理由 |
|------|------|------|
| ① **成员变量 private** | 用private修饰所有字段 | 封装性 |
| ② **提供无参构造器** | 必须有一个public无参构造 | 框架/工具需要 |
| ③ **getter/setter** | 每个private字段都提供 | 外部访问通道 |

> **💡**：IntelliJ IDEA 可以用 `alt+insert` 快捷键一键生成构造器、getter/setter。

## ⚠️ 常见考题 / 易错点

### 🎯 选择题高频考点

#### 1. 成员变量默认值
```java
// 以下哪个是正确的？
Student s = new Student();
System.out.println(s.name);  // 输出？
// A. 编译报错  B. null  C. ""
// 答案：B（引用类型默认null）
```

#### 2. 构造器相关
```java
// 题目：以下代码能否编译通过？
class Student {
    String name;
    public Student(String n) { name = n; }
}
Student s = new Student();  // ❌ 编译报错！写了有参构造后无参构造没了
```

#### 3. this的作用
```java
// 以下代码输出什么？
Student s = new Student();
s.printPass(60);
// 方法里 this.score 指的是谁的score？
// 答案：s这个对象的score（成员变量）
```

#### 4. 封装相关
```java
// private修饰的成员能被谁访问？
// A. 任何类  B. 同一个包  C. 同一个类  D. 子类
// 答案：C（只有自己类内部能访问）
```

### 🎯 编程题高频考点

| 题型 | 核心考点 | 难度 |
|------|----------|------|
| **定义JavaBean** | private+构造器+getter/setter | ⭐⭐ |
| **对象数组操作** | 多个对象存入数组，遍历操作 | ⭐⭐⭐ |
| **业务操作类** | 数据与操作分离的设计思想 | ⭐⭐⭐ |
| **电影管理系统** | 综合运用类/对象/数组/循环/switch | ⭐⭐⭐ |

### 🎯 易踩坑总结

| 坑位 | 错误 | 正确 |
|------|------|------|
| 构造器名称写错 | `public void Student() {}`（写了void） | `public Student() {}` |
| 忘了写无参构造 | 写了有参构造，new Student()报错 | 两个都写上 |
| getter拼写 | `getname()` 或 `getName()` | `getName()`（首字母大写） |
| this.xxx vs xxx | 同名时不加this指的是局部变量 | 用this区分 |
| 直接访问private | `s.score = 99` | 用 `s.setScore(99)` |

### ⭐ 面向对象设计三步走（万能模板）

```
Step 1：定义实体类（Entity/JavaBean）
        → 只存数据，private成员变量 + 构造器 + getter/setter

Step 2：定义操作类（Operator/Service）
        → 处理业务逻辑，通过构造器接收数据

Step 3：测试类（Test/Main）
        → 创建对象、组装数据、调用操作类
```

> **📝 复习建议**：面向对象是Java的核心思想！**封装**是第一个重要特性（接下来还有继承和多态）。**构造器+this+封装**是选择题高频组合。**电影管理系统**很好地演示了"实体类-操作类-测试类"的三层结构，建议手写掌握。
